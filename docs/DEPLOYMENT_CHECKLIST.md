## Deployment Day (Production Launch)

### 1-2 Hours Before Launch

1. **Final Code Review**
   - [ ] git log shows all production commits
   - [ ] No hardcoded secrets in code
   - [ ] .gitignore includes .env.production
   - [ ] package.json versions are stable (no pre-release versions)

2. **Database Preparation**
   - [ ] Prisma schema configured for PostgreSQL
   - [ ] Migration files created
   - [ ] Local test with PostgreSQL successful
   - [ ] Database backups enabled

3. **Environment Configuration**
   `ash
   # Copy template and fill in real values
   cp .env.production.example .env.production
   # Edit .env.production with:
   # - DATABASE_URL (PostgreSQL connection)
   # - PAYMENT_MODE=live
   # - STRIPE_SECRET_KEY (sk_live_...)
   # - NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY (pk_live_...)
   # - SMTP credentials
   # - ADMIN_PASSWORD and ADMIN_SECRET
   # - NEXT_PUBLIC_SITE_URL (your domain)
   `

4. **Security Final Check**
   - [ ] HTTPS enforced on domain
   - [ ] SSL certificate valid (30+ days)
   - [ ] Security headers configured
   - [ ] Admin session is httpOnly and Secure
   - [ ] No console.log of sensitive data

### Deployment - Vercel (Easiest Option)

`ash
# Push to GitHub
git add .
git commit -m \"Production deployment configuration\"
git push origin main

# Then:
# 1. Go to vercel.com ? New Project
# 2. Select GitHub repository
# 3. Add all environment variables
# 4. Click Deploy
# 5. Wait for build to complete
`

### OR Deployment - Docker

`ash
# Build Docker image
docker build -t nordsecure:latest .

# Push to registry (Docker Hub, ECR, etc.)
docker tag nordsecure:latest registry/nordsecure:latest
docker push registry/nordsecure:latest

# Deploy (via docker-compose or orchestration platform)
docker-compose -f docker-compose.yml up -d

# Run migrations
docker exec nordsecure-app npx prisma migrate deploy
`

## If Something Goes Wrong

### Rollback Procedure

`ash
# Quick rollback to previous version
git revert HEAD
git push origin main
# Vercel auto-redeploys
# OR manually trigger rollback in platform

# Verify rollback
curl https://nordsecure.eu/
`

### Database Restore

`ash
# If database corrupted
# AWS RDS: Restore from snapshot
# Supabase: Restore from automated backup
# Self-hosted: Restore from backup file

# Update DATABASE_URL if needed
# Test application reconnects
`

### Common Issues & Fixes

**\"Payment gateway not responding\"**
- Check API keys are correct and not test keys
- Verify network connectivity to payment provider
- Check payment provider status page

**\"Email not sending\"**
- Verify SMTP credentials
- Check sender domain SPF/DKIM/DMARC records
- Verify email not in spam folder
- Check email service quota

**\"Admin panel not loading\"**
- Clear browser cache
- Check ADMIN_PASSWORD environment variable
- Verify admin session cookie stored
- Check server logs for errors

**\"Orders not appearing\"**
- Check DATABASE_URL is correct
- Verify database connection working
- Run Prisma migrations: \
px prisma migrate deploy\
- Check database has tables

**\"HTTPS certificate error\"**
- Wait for DNS propagation (up to 24-48 hours)
- Manually trigger SSL certificate generation
- Check domain is correctly configured
