# ----------------------------------------------------------------------------
# DEPLOYMENT GUIDE - Quick Start
# ----------------------------------------------------------------------------

This file contains quick instructions for deploying NordSecure to production.

## PRE-DEPLOYMENT CHECKLIST

Before deploying to production, ensure you have:

1. Payment Provider Accounts
   - Stripe account with live keys
   - BTCPay instance (optional)

2. Email Service
   - SendGrid, Postmark, or AWS SES account
   - Verified sender domain/email

3. Database
   - PostgreSQL database created (AWS RDS, Supabase, or self-hosted)
   - Database credentials saved securely

4. Hosting Platform
   - Account created (Vercel, AWS, DigitalOcean, etc.)
   - GitHub repo connected (if using auto-deploy)

5. Domain & SSL
   - Domain registered
   - DNS records ready
   - SSL certificate (auto-generated by most platforms)

6. Admin Credentials
   - Strong ADMIN_PASSWORD (32+ characters)
   - Strong ADMIN_SECRET (for session signing)

## DEPLOYMENT OPTIONS

Option A: Vercel (Recommended - Easiest)
? Perfect for Next.js apps, auto-deploy from GitHub
? See docs/PRODUCTION_SETUP.md for step-by-step

Option B: Docker (Self-Hosted)
? Full control, works on any VPS or cloud provider
? docker-compose up -d to start

Option C: AWS ECS
? Enterprise-grade, auto-scaling capabilities
? Requires AWS account and Docker knowledge

## POST-DEPLOYMENT VERIFICATION

After deployment, verify:

1. Homepage loads: https://nordsecure.eu/
2. Admin page loads: https://nordsecure.eu/admin/login
3. Admin login works (password from ADMIN_PASSWORD env var)
4. Create test order and verify email sent
5. Check order in admin panel
6. Update order status and verify email to customer

## PRODUCTION SECURITY CHECKLIST

Before going live:

- HTTPS enforced (redirect HTTP to HTTPS)
- ADMIN_PASSWORD changed from default
- ADMIN_SECRET is 32+ random characters
- All Stripe keys are LIVE keys (sk_live_*, pk_live_*)
- Database password is 32+ random characters
- Email credentials verified and working
- No secrets committed to Git
- Database backups configured
- SSL certificate valid and auto-renewing

## MONITORING

After deployment, monitor:
- Error tracking: Sentry or provider logs
- Uptime: UptimeRobot
- Database: Provider's monitoring dashboard
- Email delivery: SendGrid/Postmark dashboard
- Payment processing: Stripe dashboard
